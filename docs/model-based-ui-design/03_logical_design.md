# フェーズ3: デザイン - 構造からUIへの変換

## 1. 論理UIデザイン (抽象プロトタイピング)

モデルを具体的な画面構成要素にマッピングします。
本アプリケーションは「シングルページ」で完結する構成とします。

### 画面構成要素 (ワイヤーフレーム定義)

画面を大きく「操作エリア（入力）」と「プレビューエリア（出力）」の2つに分割します。

#### A. ヘッダー / 操作エリア
ユーザーがパターンを選択するための領域。

*   **App Title**: アプリケーション名。
*   **Pattern Selector**: 1〜12のパターンを選択するUI。
    *   モバイルファーストを考慮し、タップしやすい大きさのボタンを配置。
    *   12個のボタンをグリッド状、または横スクロールで配置する。
    *   現在選択中の番号を視覚的に強調する。

#### B. プレビューエリア (鑑定書雛形)
印刷されるA4用紙のレイアウトを画面上で再現する領域。
このエリアはCSSの `@media print` でそのまま印刷される対象となる。

1.  **タイトル・基本情報欄 (手書き用)**
    *   タイトル：「鑑定書」などの表記。
    *   氏名記入欄（空欄、下線）
    *   生年月日記入欄（空欄、下線）
2.  **エネルギーが落ちる年 (自動表示)**
    *   `Pattern.EnergyLowYear` を表示。
3.  **エネルギーUP色 (自動表示 + 手書き)**
    *   **守護色**: 手書き用の枠（タイトル「守護色」＋空欄）。
    *   **その他6色**: `Pattern.EnergyUpColors` の各色（自分色、やる気色...）を表示。
        *   色名（テキスト）と色見本（円形や四角形の塗りつぶし）をセットで配置。
4.  **タブー色 (自動表示)**
    *   `Pattern.TabooColor` を表示（色名＋色見本）。
5.  **インテリアアドバイス (自動表示)**
    *   `Pattern.InteriorAdvice` のテキストを表示。読みやすい行間を確保。
6.  **その年のラッキー・タブーカラー (手書き用)**
    *   年記入欄（空欄）
    *   ラッキーカラー記入欄（空欄）
    *   タブーカラー記入欄（空欄）

#### C. フッター / アクションエリア
*   **Print Button**: 印刷ダイアログを呼び出すボタン。
    *   モバイルでは画面下部に固定（Sticky）配置し、いつでも押せるようにする。

### レイアウト・レスポンシブ方針

*   **モバイルビュー (画面幅狭)**:
    *   上部にPattern Selector（2行6列や3行4列、あるいは横スクロール）。
    *   その下にプレビューエリア全体を縮小表示（`transform: scale()` 等を使用し、A4比率を維持したまま画面幅に収める）。
*   **PCビュー (画面幅広)**:
    *   左カラム：Pattern Selector（縦並びやグリッド）。
    *   右カラム：プレビューエリア（実寸に近いサイズで表示）。
*   **印刷ビュー**:
    *   Pattern Selector、Print Button、ヘッダー等は全て非表示（`display: none`）。
    *   プレビューエリアのみをA4用紙1枚にフィットさせて出力。
