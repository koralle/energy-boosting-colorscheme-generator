# フェーズ1: 分析 - 領域と要求の理解

## 1. ユーザーシナリオ / 業務フロー

本アプリケーションを利用する「占い師」の業務フローと、システムが関与する部分（システム境界）を定義します。

### 業務フロー概要
1. **事前準備 (システム外)**: 占い師は依頼者から「本名」「生年月日」を受領し、独自のロジックで依頼者に該当する「パターン番号 (1〜12)」と「守護色」を特定する。
2. **雛形作成 (システム内)**: 占い師は本アプリケーションにアクセスし、特定した「パターン番号」を選択する。
3. **確認・印刷 (システム内)**: アプリケーションは選択されたパターンに基づいて鑑定書の雛形を表示する。占い師は内容を確認し、印刷（またはPDF保存）を実行する。
4. **仕上げ (システム外)**: 印刷された鑑定書に対し、占い師は「依頼者名」「生年月日」「守護色」「その年のラッキーカラー・タブーカラー」を手書きで記入し、完成させる。
5. **提供 (システム外)**: 完成した鑑定書を依頼者に渡し、解説を行う。

### ユーザーストーリー
* **占い師として**、手軽に綺麗に印刷できる鑑定書の雛形が欲しい。なぜなら、毎回手書きで全ての項目を作成するのは時間がかかり、見た目の説得力もバラつくからだ。
* **占い師として**、パターン番号を選択するだけで必要な情報（色の組み合わせやアドバイス）が自動的に埋まってほしい。なぜなら、これらの組み合わせは複雑で転記ミスが発生しやすいからだ。

## 2. 用語集（ドメイン辞書）

業務で扱われる用語とシステム上での意味を定義します。

| 用語 | 英語名 (Code Name) | 定義・説明 |
| :--- | :--- | :--- |
| **占い師** | `FortuneTeller` | 本システムの利用者。鑑定を行う主体。 |
| **依頼者** | `Client` | 占いを依頼する人。鑑定書の受け取り手。 |
| **鑑定書** | `AppraisalReport` | 最終的な成果物。エネルギーUP色やアドバイスが記載されたドキュメント。 |
| **鑑定書の雛形** | `ReportTemplate` | 本システムが出力する、一部項目（鑑定パターン依存項目）が埋め込まれた印刷物。手書き欄は空白となっている。 |
| **鑑定パターン** | `AppraisalPattern` | 1〜12の番号で管理される鑑定結果の分類。これにより「エネルギーUP色（守護色以外）」「エネルギーが落ちる年」「タブー色」「インテリアアドバイス」が一意に定まる。 |
| **エネルギーUP色** | `EnergyUpColor` | 依頼者にとって縁起のいい色の総称。7種類（守護色＋その他6種）ある。 |
| **守護色** | `GuardianColor` | エネルギーUP色の一つ。鑑定パターンに依存せず、個別に特定される色。システムでは「手書き」項目として扱う。 |
| **自分色** | `SelfColor` | 鑑定パターンによって決まるエネルギーUP色の一つ。自分の魅力を引き出す色。 |
| **やる気色** | `MotivationColor` | 鑑定パターンによって決まるエネルギーUP色の一つ。やる気が出る色。 |
| **精神安定色** | `StableColor` | 鑑定パターンによって決まるエネルギーUP色の一つ。精神を安定させる色。 |
| **決断色** | `DecisionColor` | 鑑定パターンによって決まるエネルギーUP色の一つ。決断時に良い色。 |
| **健康色** | `HealthColor` | 鑑定パターンによって決まるエネルギーUP色の一つ。健康に導く色。 |
| **経済色** | `EconomicColor` | 鑑定パターンによって決まるエネルギーUP色の一つ。金運に関する色。 |
| **エネルギーが落ちる年** | `EnergyLowYear` | 運気が下がるとされる時期（例：子、丑、-）。鑑定パターンによって決まる。 |
| **タブー色（薬色）** | `TabooColor` | アンラッキーカラーだが、エネルギーが落ちる年には「薬色」として使える色。鑑定パターンによって決まる。 |
| **インテリアアドバイス** | `InteriorAdvice` | 能力を引き出すためのインテリアに関する助言。鑑定パターンによって決まる12種類のテキスト。 |
| **ラッキーカラー/タブーカラー(年)** | `YearlyLuckyTabooColor` | その年ごとのラッキーカラーとタブーカラー。鑑定パターンに依存せず、個別に特定される。システムでは「手書き」項目として扱う。 |
