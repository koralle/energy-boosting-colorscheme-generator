# Ten Usability Heuristics

ヤコブ・ニールセンの10のユーザビリティヒューリスティックを使用してUIを評価・分析する。

## 評価ワークフロー

1. **UIの把握**: 提供されたスクリーンショット、コード、またはブラウザスナップショットからUI構造を理解する
2. **ヒューリスティックの確認**: [references/heuristics.md](references/heuristics.md) を参照して各ヒューリスティックを確認する
3. **評価実施**: 各ヒューリスティックに基づいてUIを分析し、違反や改善点を特定する
4. **優先度の付与**: 見つかった問題を深刻度（低・中・高・クリティカル）で分類する
5. **改善提案**: 各問題に対する具体的な改善案を提示する

## 優先度の基準

| 優先度 | 定義 | 例 |
|--------|------|-----|
| クリティカル | ユーザーがタスクを完了できない、データ損失のリスク | 保存ボタンがない、取り消しできない削除 |
| 高 | 大幅なユーザビリティ低下、多くのユーザーが困る | エラーメッセージが不明、ナビゲーションが見つからない |
| 中 | 一部のユーザーに影響、迂回策がある | 用語が不統一、小さな配置の問題 |
| 低 | 軽微な問題、理想的な状態からの逸脱 | 微妙な色の不統一、小さな余白の調整 |

## 出力形式

評価結果は以下の形式で出力する：

### 評価サマリー
- 全体的なユーザビリティの評価
- 主要な問題点の概要

### 問題点リスト（優先度順）
各問題点について以下を含める：
1. **該当ヒューリスティック**: 違反しているヒューリスティック
2. **問題の説明**: 何が問題か
3. **深刻度**: クリティカル/高/中/低
4. **改善提案**: 具体的な解決策

### 改善提案の優先リスト
直ちに対応すべき項目をトップにリストアップする
